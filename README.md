**PROBLEM STATEMENT**
Students and beginners in electronics often choose LED resistors by guessing or copying values from other circuits. This can cause the LED to draw too much current or appear too dim. A simple software tool is needed to calculate the correct resistor value and provide basic safety feedback before the circuit is built.
**AIM OF THE PROJECT**
Students and beginners in electronics often choose LED resistors by guessing or copying values from other circuits. This can cause the LED to draw too much current or appear too dim. A simple software tool is needed to calculate the correct resistor value and provide basic safety feedback before the circuit is built.
**OBJECTIVES OF THE PROJECT**
.Accept user inputs for LED label/colour, circuit supply voltage, LED forward voltage, and desired current in milliamperes
.Enforce strict input validation rules to reject negative electrical parameters, zero current, and physically impossible circuit loops where the supply voltage is less than or equal to the forward voltage
.Apply Ohm's Law and Joule's Law sequentially to calculate resistance and power values.
.Classify configurations into distinct statuses (Design Accepted, Warning, or Invalid Design).
.Output results to the console window and compile them into `led_resistor_report.txt`.

**EXPLANATION OF THE LED RESISTOR CALCULATION FORMULA**
The application applies standard electrical engineering laws sequentially using the following text-based steps:

1. **Current Conversion:** The target LED current value is initially received from the user in milliamperes. To prepare this value for standard electrical equations, the system converts it into Amperes by taking the desired LED current in milliamperes and dividing it by one thousand.

2. **Ohm's Law for Calculating Resistance:** To determine the exact size of the current-limiting resistor required, the system calculates the total voltage drop that the resistor needs to handle. This is found by taking the circuit supply voltage and subtracting the LED forward voltage. The resulting voltage difference is then divided by the previously converted LED current in Amperes to give the recommended resistor value in ohms.

3. **Joule's Law for Power Dissipation:** To ensure the physical resistor can safely handle the heat generated by the circuit without burning out, the minimum resistor power rating in watts is calculated. This is found by taking the LED current in Amperes, multiplying it by itself to square it, and then multiplying that result by the calculated recommended resistor value in ohms.

**ALGORITHM AND PSEUDOCODE SUMMARY**

**ALGORITHM SUMMARY**
The program initializes an outer execution loop based on the total number of circuits specified by the user. For each loop iteration, the system prompts for input data and routes it through an input validation decision block. If the electrical values are mathematically or physically invalid, the program logs an "Invalid Design" status string and skips all calculation blocks to jump directly to the loop condition evaluation. If the inputs are valid, the program processes the conversion, resistance, and power formulas in sequential order, checks the current against a thirty-milliamp safety threshold to append the correct status message, and prints the result to the console window. Once all specified circuits have completed their turn through the loop, the system executes file-writing streams to save a complete evaluation summary to an external plain text file.

**PSEUDOCODE SUMMARY**
IF (supplyVoltage <= forwardVoltage) OR (ledCurrentMilliAmps <= 0) THEN
    SET designStatus = "Invalid Design"
ELSE
    CALCULATE ledCurrentAmps = ledCurrentMilliAmps / 1000
    CALCULATE resistorValue = (supplyVoltage - forwardVoltage) / ledCurrentAmps
    CALCULATE resistorPower = ledCurrentAmps * ledCurrentAmps * resistorValue
    
    IF (ledCurrentMilliAmps > 30) THEN 
        SET designStatus = "Warning: LED current may be too high for a standard LED."
    ELSE 
        SET designStatus = "Design Accepted"
    END IF
    
    DISPLAY resistorValue, resistorPower, designStatus
END IF
